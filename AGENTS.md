# Spotted Server

Go + MySQL + Chi router. Code-judge platform with quiz participation and keystroke tracking.

## Setup

```bash
cp .copy.env .env
docker compose . up -d      # starts MySQL
chmod +x ./scripts/migrateup.sh
./scripts/migrateup.sh      # runs schema + trigger migrations
chmod +x ./scripts/seed.sh
./scripts/seed.sh           # populates seed data
air                          # runs dev server (PORT from .env, default 69420)
```

## Commands

```bash
go test ./...                              # unit tests
go test ./path/to/pkg -run TestName       # single test
go test ./... -cover                      # with coverage (CI runs this)
gosec ./...                               # security scanning
go fmt ./... && staticcheck ./...         # style check (CI order)
```

## Architecture

- `cmd/api/main.go` — server entrypoint
- `internal/server/` — HTTP handlers, split by feature (profiles, quizes, companies, codejudge)
- `internal/database/` — **generated by sqlc**, do not edit manually; regenerated from `seeders/sqlc.yaml`
- `internal/auth/` — JWT auth logic
- `sql/schema/` + `sql/triggers/` — goose migrations (schema first, then triggers)
- `seeders/` — separate Go module for database seeding

## Key quirks

- Migrations run in two steps: schema migrations, then trigger migrations. Always run both.
- `scripts/migrateup.sh` and `scripts/migratedown.sh` source `.env` before running goose.
- `seeders/` builds its own binary (`go build -o seeder` inside `seeders/`) and runs it independently.
- `sqlc` regenerates `internal/database/*.go` from `seeders/sqlc.yaml` + `sql/schema/`. Re-run codegen after schema changes.
- Some static files are gitignored (`*.js`, `*.css` under `internal/server/static/`) and expected to be generated or served from elsewhere.
- CI uses Go 1.21.5; local toolchain is 1.23.x.
